instalation guide for elementary OS

ps: *user* should be replaced by the user name, normally the same used when logging in to the computer.

Clone the github.com/tbfraga/COPSolver/ directory to your computer inside some folder named GitHub.

in /home/*user*/ create the Documents folder if it doesn't exist.

copy COPSolver folder that is located on GitHub/COPSolver/Documents/ inside the Documents folder you just created.

PS: following steps needs you to have adm access.

copy COPSolver folder that is located on GitHub/COPSolver/install/opt/ inside the /opt/ folder.

create a symbolic link for accessing /opt/COPSolver/COPSolver inside /bin/ folder.

create a symbolic link for accessing /opt/COPSolver/COPSolver inside /usr/bin/ folder.

copy COPSolver.desktop that is located on GitHub/COPSolver/install/usr/share/applications/ inside /usr/share/applications/ folder.

Ready !!! You can now access COPSolver from the elementary OS Applications menu :)

PS: Installation on other versions of ubuntu can be done through the same procedure, however the COPSolver.desktop file must be adjusted to use the appropriate terminal.

using COPSolver on elementary OS

Once the solver has been installed, just click on the Applications menu and then on the COPSolver icon.

(1) Size the terminal window if you want and then press enter.

(2) Type 1, 2 or 3 to choose the problem definition method (as explained bellow) and then press enter.

Depending on the chosen method, COPSolver will work differently.

If you type 1:

COPSolver will come up with an optimal solution for the problem specified in the data.txt file.

If you type 2:

(3) Type 1, 2 or 3 to select one of the solver's predefined problems.

COPSolver will then come up with an optimal solution for the specified problem.

If you type 3:

(3) Enter an interger source value to generate a random problem and press enter.

(4) Enter an integer value to determine the number of products in the batch and press enter.

COPSolver will then come up with an optimal solution for the specified problem.

using data.txt for defining a benchmark

If you use standart installation, data.txt will be located at /Ä§ome/*user*/Documents/COPSolver/data.txt

you can change data.txt file as specified below:

first line: specify only one integer value for informing the number of products (N);

second line: specify N integer values for informing the production rate of each product;

third line: specify N integer values for informing the demand of each product;

fourth line: specify only one integer value for informing the outlets total inventory available;

fifth line: specify N integer values for informing the outlets inventory available for each product;

sixth line: specify only one integer value for informing the factory total inventory available;

seventh line: specify N integer values for informing the factory inventory available for each product;

eighth line: specify only one integer value for informing the multi-product batch processing time limit.

outputs

COPSolver will generate the following outputs:

output.txt: a file containing the last solver solution;

problem.txt: a file containing the last problem definition;

data.ldt: a LINGO data file with last problem definition for verifying the solution found by COPSolver;

output.txt problem.txt files will be located at /home/work/Documents/COPSolver/;

data.ldt file will be located at /home/work/Documents/COPSolver/LINGOSolver/MPBPTMP/.

LINGO solver

Inside the /home/work/Documents/COPSolver/LINGOSolver/MPBPTMP/ folder there is a code in LINGO to solve the mathematical model developed for the muti-product batch processing time maximization problem.
